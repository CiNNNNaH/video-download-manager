# Portable Release Guide

## Goal
Produce a portable bundle that is repeatable, clean, and safe to hand to another machine without carrying stale runtime state.

## Recommended flow
1. bootstrap Python dependencies
2. run pretest checks
3. run regression suite
4. run multi-site validation
5. build the portable folder with your normal PyInstaller / onedir flow
6. clean runtime artifacts
7. verify the bundle
8. create the release zip
9. smoke test the extracted zip on a clean folder

## Commands
```bat
scripts\build_portable_release.bat
scripts\clean_runtime_artifacts.bat
scripts\verify_portable_bundle.bat <portable_folder>
scripts\package_release_bundle.bat <portable_folder>
```

## Required portable outputs
- portable application folder
- `log.txt` in bundle root
- empty `logs/` folder
- `release_manifest.json` generated by verification
- `README.md`, `CHANGELOG.md`, `LICENSE`
- `docs/` folder

## Bundle acceptance rules
- verification must pass before zipping
- `logs/` must be empty
- `data/history.json` must be empty or absent
- `data/settings.json` must not carry machine-specific paths that will confuse another user
- no `__pycache__`, `.pyc`, or stale build leftovers in the shipped folder
- zip file name should remain ASCII-only

## Do not call it release-ready unless
- pretest passed
- regression suite passed
- multi-site manual sweep was performed
- at least one public YouTube download passed
- at least one non-YouTube site family passed
- packaged build opens on the target Windows machine after extraction
